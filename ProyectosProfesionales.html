Python

Optimización del Programa de Predicciones en Python

Durante mi experiencia laboral, me enfoqué en mejorar la eficiencia y funcionalidad del programa utilizado para la creación de predicciones de demanda. Este programa, originalmente desarrollado por mi predecesor en el cargo (Carlos Yebgeni Pierdan), empleaba modelos de machine learning, principalmente derivados del árbol de decisiones, para prever la demanda de una amplia gama de productos. Este era un desafío significativo debido a la diversidad de productos vendidos, que ascendió a aproximadamente 3800 diferentes productos en 2023.

El programa opera filtrando y limpiando la información histórica de ventas, organizándola por mes y año, y gestionando valores nulos. Posteriormente, se aplica un filtro lógico para determinar si el producto tiene al menos un año de registro de ventas, antes de proceder con la predicción.

Una vez pasado este filtro, se definen los meses de prueba para validar los modelos de machine learning y la cantidad de meses futuros para la predicción. Se calculan los retrasos o lags de la serie de tiempo para evaluar la autocorrelación y se selecciona el mejor retraso, generalmente dentro de un rango de seis meses.

Se crean listas de funciones por modelo de machine learning, cada una aplicando una transformación diferente a los datos para mejorar las predicciones. Estas funciones se aplican a los datos y se comparan entre sí mediante su error promedio, obtenido de los meses de prueba.

Finalmente, se selecciona la mejor función y se implementa, junto con la técnica de bootstraping para mejorar los intervalos de confianza. Se genera una gráfica que muestra la serie de tiempo del producto, la estacionalidad y los puntos atípicos de la serie, proporcionando una herramienta integral para la toma de decisiones en la gestión de inventarios.

El programa originalmente requería entre 13 y 16 minutos para ejecutar los modelos, seguidos de una tarea adicional de copiar la información a un documento de Excel. Durante mi evaluación, identifiqué varias deficiencias en el programa, siendo la principal el hecho de que algunas funciones se ejecutaban hasta tres veces, lo cual consumía la mayor parte del tiempo de ejecución. Para abordar este problema, optimicé el código reciclando la información para diferentes pasos y creando funciones adicionales para mejorar la legibilidad y eficiencia del programa.

Además, dado el gran número de productos que debían procesarse, agregué una función que extraía un archivo CSV con una lista de IDs de productos. Esto permitía ejecutar los modelos para múltiples productos simultáneamente, con la información descargada en una carpeta para su análisis posterior o para compartir según fuera necesario.

Estas mejoras redujeron significativamente el tiempo de ejecución del programa a aproximadamente 3 minutos por producto. Como resultado, pude obtener las predicciones de más de 140 productos en un solo día, lo que anteriormente habría requerido una semana de trabajo dedicado exclusivamente a esa tarea. Esta optimización me permitió gestionar mis responsabilidades de manera más eficaz, equivaliendo a dos semanas de trabajo en un solo día.
